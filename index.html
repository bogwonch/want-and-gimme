    Want and Gimme

    • want document.tex: go figure out how to build me my document.
    • gimme pdf: work out if there is anything you can convert into a pdf and do so.

    When I'm working on a project I can't remember what all the build commands are so I use a Makefile. Wherever possible I use implicit rules to just convert whatever I've called my code into the obviously named other format.

    Some of these come baked into Make through a config file hidden somewhere not documented in my manpages. Screw this: just gimme what I want.

    Usage

    Rules go in ~/.makerules. Normal makefile syntax we all know and hate. want will pick up this file and use it to try and build the files you want.

    The gimme script is for when you're even more lazy and rude or you have a bunch of files that all need converting (like when you're writing .dot files then converting them to .eps to stick in a LaTeX document). Just tell it to gimme filetype and it will try and convert any file it knows how to into that filetype.

    Options

    Some want rules are customizable! You wanna convert a graphviz diagram but just can't decide which layout engine to use? We've got you covered! Use the GV variable to set the layout engine.

    Wanna see what other options are available? Just run want without any arguments.

    Do you want to be really cool and add more options? They're written in your .makerules file.

    GV ?= dot # Sets the GraphViz layout engine OptionName ?= DefaultValue # Documentation
